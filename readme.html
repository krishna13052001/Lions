API documentation

Login:
    Fields => id, password
    Checks => whether the id is present in 'users' table, the match the encrypted(BLOWFISH) password
    Method => POST, url: https://lions3234d2.com/api.php
    API Call => submit the form with a field 'login':'true'
    Response => details:[array with member details], error:'success' or 'error', type:'user' or 'invalid'
    
User Verify:
    Fields => id, regionName, zoneName, firstName, middleName, lastName, address1, address2, city, state, postalCode, email, phone, spouseName, dob, gender, occupation
    Checks => middleName, spouseName are optional
    Method => POST, url: https://lions3234d2.com/api.php
    API Call => submit the form with a field 'update-profile':'true'
    Response => status:'error' or 'success'

Add event:
    Fields => eventTitle, amount, chiefGuest, venue, date(optional), description, eventType, authorId, clubId, image[]
    Checks => check image extension, and only allow the following: 'jpg', 'png', 'jpeg', 'gif', 'tif'
    Method => POST, url: https://lions3234d2.com/api.php
    API Call => submit the form with a field 'add-event':'true'
    Response => status: 'error' or 'success', imgError: [files with invalid extension]'
    Note: Images are compressed automatically and stored in 'images' table with category: 'news' or 'events' or 'activities' and categoryId as foreign keys
